## 생산(N~W) / 출고(X~AJ) 셀 값 형식

두 가지 형식이 **같은 컬럼 내에 혼재**함.

| 형식 | 생산 비율 | 출고 비율 | 예시 | 의미 |
|------|-----------|-----------|------|------|
| `M/D NNN조` | ~27% | ~4% | `"3/11 503조"` | 날짜 + 수량 통합 (최신 방식) |
| `MM월 DD일` | ~73% | ~95% | `"01월 13일"` | 날짜만 기록, 수량 없음 (구형 방식) |
| 기타 | ~0% | ~0% | `"완료"`, `"카톡"`, `"생산완료"` | 자유 메모, 파싱 불가 |

> `M/D NNN조` 형식에서 연도는 없으며, 의뢰번호 앞 2자리(`YY`) 기반으로 추론 필요.

## 파싱 시 주의사항

1. **중간 헤더 행 필터링**
   - Row 3178 등 A컬럼 값이 `"의뢰번호"` 텍스트인 행은 데이터 행이 아님

2. **같은 의뢰번호 복수 행**
   - `doc_no`(`YY-NNNN`)가 같은 행이 여러 개 존재 → 라인 아이템 분리
   - DB의 `glassflow_order_docs`(1) : `glassflow_order_items`(N) 관계에 대응

3. **의뢰수량 공백 trim**
   - `" 442 "` → `"442"` → `parseInt()` 처리 필요

4. **날짜 연도 추론**
   - K, L, M 컬럼의 `MM월 DD일` 형식에 연도 없음
   - 생산/출고의 `M/D` 형식도 연도 없음
   - 의뢰번호 앞 2자리(`YY`)를 기준으로 연도 결정

5. **날짜 컬럼 자유텍스트 처리**
   - K, L, M 컬럼에 `"미정"`, `"ASAP"`, `"급발주"`, 담당자명 등 비날짜 텍스트 포함 → null 처리

6. **H(TPS) 컬럼 vs K(주문서도착일) 컬럼**
   - 22~23년 초기: H 컬럼에 `MM/DD/YY` 형식으로 주문서 도착일 기록
   - 24년 이후: K 컬럼에 `MM월 DD일` 형식 사용
   - 마이그레이션 시 둘 중 하나를 우선순위로 선택 필요

7. **E~G 컬럼 의미 불확정**
   - 규격 정보(`"12단열(S)"`, `" 5CL+5로이"`)와 폐기/보류 메모(`"25.10.22 폐기"`)가 혼용
   - 채움률이 낮아(25~18%) 선택 입력 필드로 취급

8. **생산/출고 수량 파싱**
   - `"3/11 503조"` → 날짜: `3/11`, 수량: `503`
   - `"MM월 DD일"` → 날짜만, 수량: 알 수 없음 (의뢰수량으로 대체 또는 null)
   - 쉼표 포함 숫자 처리 필요: `"1,368조"` → `1368`

## 데이터베이스 스키마 연계

### 테이블 관계도

```
발주현황 1행 (같은 doc_no)
    │
    ├─► glassflow_order_docs (1행, doc_no UNIQUE)
    │       id, doc_no, customer, site_name
    │       request_date, due_date, tps_date, arrival_date
    │       source='excel', created_at, updated_at
    │
    └─► glassflow_order_items (1행 per 엑셀 행)
            id, doc_id → order_docs.id
            item_name, order_qty, area_m2
            (width_mm, height_mm, location: PDF 방식에서만 사용)
                │
                ├─► glassflow_production_logs (N행)
                │       item_id → order_items.id
                │       seq, produced_date, produced_qty
                │       is_completed, note
                │
                └─► glassflow_shipment_logs (N행)
                        item_id → order_items.id
                        seq, shipped_date, shipped_qty, note
```

### 엑셀 컬럼 → DB 필드 매핑

#### `glassflow_order_docs`

| 엑셀 열 | 엑셀 헤더 | DB 필드 | 변환 처리 |
|---------|-----------|---------|-----------|
| A | 의뢰번호 | `doc_no` | trim. UNIQUE → upsert(onConflict: doc_no) |
| B | 거래처 | `customer` | trim |
| C | 현장명 | `site_name` | trim. null이면 행 스킵 |
| H | TPS | `tps_date` | `MM/DD/YY` → `YYYY-MM-DD`. 비날짜 텍스트는 null |
| K | 주문서도착일 | `arrival_date` | `MM월 DD일` → `YYYY-MM-DD` (연도: doc_no 앞 2자리) |
| L | 생산의뢰일 | `request_date` | 동일 |
| M | 납품요청일 | `due_date` | 동일. `"미정"` 등 → null |
| — | — | `source` | 고정값 `'excel'` |

#### `glassflow_order_items`

| 엑셀 열 | 엑셀 헤더 | DB 필드 | 변환 처리 |
|---------|-----------|---------|-----------|
| G | 품명/비고2 | `item_name` | trim. null 허용 |
| D | 의뢰수량 | `order_qty` | 숫자만 추출(`replace(/[^0-9]/g, '')`) |
| E | 면적 | `area_m2` | 현재 마이그레이션 스크립트에서 미사용 |
| — | — | `width_mm`, `height_mm`, `location` | PDF 업로드 방식에서만 사용, 엑셀 마이그레이션 시 null |

#### `glassflow_production_logs` (N~W, 최대 10회)

| 엑셀 셀 값 형식 | `produced_date` | `produced_qty` | `is_completed` | `note` |
|----------------|-----------------|----------------|----------------|--------|
| `"M/D NNN조"` | `YYYY-MM-DD` 파싱 | 숫자 추출 | false | null |
| `"MM월 DD일"` | `YYYY-MM-DD` 파싱 | `order_qty` 대입 | false | null |
| `"완료"` | null | `order_qty` 대입 | **true** | null |
| 숫자 문자열 | null | 숫자 | false | null |
| 기타 텍스트 | null | null | false | 원문 저장 |
| null/빈값 | — | — | — | 행 스킵 |

#### `glassflow_shipment_logs` (X~AJ, 최대 13회)

생산 로그와 동일한 파싱 로직. `shipped_date`, `shipped_qty`, `note` 필드에 저장.

### `glassflow_item_status` 뷰

4개 테이블을 통합하는 뷰로, 직접 수정 불가. 계산 필드:

| 뷰 필드 | 계산 방법 |
|---------|-----------|
| `total_produced_qty` | `SUM(production_logs.produced_qty)` |
| `total_shipped_qty` | `SUM(shipment_logs.shipped_qty)` |
| `pending_qty` | `order_qty - total_shipped_qty` |
| `status` | `pending` / `in_progress` / `produced` / `partial` / `shipped` |

## 마이그레이션 스크립트 처리 흐름

```
scripts/migrate-excel.ts

1. Excel 행 순회 (fromRow ~ limit)
   ├─ 스킵 조건: docNo 없음, docNo === '의뢰번호'(헤더 행), site_name 없음
   ├─ 필터: --doc-no=, --years=, --from=, --limit=
   │
2. order_docs upsert (onConflict: doc_no)
   └─ 같은 doc_no 여러 행이 와도 docs는 1건만 유지

3. order_items insert
   └─ 같은 doc_id의 첫 행 처리 시 기존 items 전체 삭제 후 재삽입

4. production_logs insert
   └─ item_id 기준으로 삭제 후 qty !== null인 항목만 삽입

5. shipment_logs insert
   └─ 동일
```

### 실행 명령

```bash
# 드라이런 (DB 저장 없이 파싱 결과 확인)
npx tsx scripts/migrate-excel.ts --dry-run

# 전체 실행
npx tsx scripts/migrate-excel.ts --run

# 특정 연도만
npx tsx scripts/migrate-excel.ts --run --years=25,26

# 특정 의뢰번호 재처리
npx tsx scripts/migrate-excel.ts --run --doc-no=25-0190
```
