## REQ-01. 고급 검색에 비고 필드 포함

**요청 내용**
대시보드 검색 시 비고(엑셀 E~G 컬럼) 내용을 검색 대상에 포함해달라.
아파트 동 관련 정보가 비고란에 기재되어 있어, 이를 검색해서 조회해야 하는 업무 필요성이 있음.

**현황**
- `/api/items` 검색은 `site_name`, `customer`, `doc_no` 기준으로만 동작
- `glassflow_order_items.item_name` (엑셀 G열)은 DB에 저장되어 있으나 검색 대상 외
- 엑셀 E열(면적), F열(간봉/메모)은 현재 DB에 미저장

**구현 방향**
- `glassflow_order_items`에 `note` 또는 `remark` 컬럼 추가 (엑셀 F, G열 통합 저장)
- `/api/items` 검색 파라미터에 `keyword` 추가 → `item_name`, `note`, `site_name` 대상 ILIKE 검색
- 대시보드 검색 UI에 키워드 입력창 추가

## REQ-02. 엑셀 파일 업로드

**요청 내용**
현재 PDF만 업로드 가능한데, 엑셀 파일도 업로드하여 데이터를 등록할 수 있게 해달라.

**현황**
- `/upload` 페이지는 PDF → GPT-4o Vision → 파싱 파이프라인만 지원
- 엑셀 마이그레이션은 `scripts/migrate-excel.ts` 스크립트로만 가능 (CLI, 개발자 전용)
- `xlsx` 패키지는 이미 프로젝트에 설치되어 있음

**구현 방향**
- `/upload` 페이지에 엑셀 업로드 탭 추가 (`.xlsx`, `.xls` 허용)
- 업로드된 파일에서 `발주현황` 시트를 자동 인식하여 파싱
- 파싱 결과를 UI 테이블로 미리보기 → 사용자 확인 후 저장
- 파싱 로직은 `scripts/migrate-excel.ts`의 `parseLogCell`, `parseExcelDate` 재사용
- API: `POST /api/upload-excel` 신규 추가

## REQ-03. 사용자 인증 및 회원가입 (자체 판단 추가)

**배경**
공사관리부 직원 전체 오픈이 예정되어 있으나, 현재 시스템에 인증이 없어 URL만 알면 누구나 접근 가능. 다수 사용자 환경에서 데이터 보호 및 접근 제어가 필요.

**현황**
- 인증 없음. 모든 페이지 및 API 엔드포인트 공개 접근 가능
- Supabase가 이미 도입되어 있어 Supabase Auth 활용 가능
- 현재 `supabase-admin.ts`는 service role key 사용 (RLS 우회) → 인증 도입 시 RLS 정책 설계 필요

**구현 방향**
- Supabase Auth (이메일/비밀번호) 기반 인증 도입
- 회원가입 페이지 (`/auth/signup`): 이름, 이메일, 비밀번호
- 로그인 페이지 (`/auth/login`)
- 미인증 사용자 → `/auth/login` 리다이렉트 (Next.js middleware 적용)
- 역할 구분 검토: 관리자(업로드·수정 가능) / 일반 직원(조회만 가능)
- `@supabase/ssr` 패키지 이미 설치되어 있어 서버/클라이언트 세션 관리 가능
